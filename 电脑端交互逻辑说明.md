# 电脑端交互逻辑说明

## 1. 左侧栏目

左侧只保留两个栏目：

- 项目
- 导入导出

不单独设置：

- 样品
- 备份记录

样品管理能力合并到“项目”页面内完成，备份记录可放在导入导出页面内作为辅助信息。

## 2. 项目页面核心结构

项目页面分为三个区域：

```text
左侧项目列表
→ 中间样品列表/样品详情
→ 右侧选中样品信息
```

推荐页面结构：

```text
顶部：搜索、筛选、导入手机包、导出给手机、导出 Excel
左侧：项目列表
中间：选中项目下的样品列表
右侧：选中样品详情
```

## 3. 主交互流程

核心流程：

```text
项目列表
→ 样品详情
→ 弹出窗口项目内容调整
```

说明：

- 用户先在项目列表中选择项目。
- 系统在右侧展示该项目下的样品。
- 用户点击某个样品后，右侧显示样品详情。
- 用户可以直接编辑样品内容并保存。
- 用户点击“调整项目”时，弹出窗口修改项目内容。
- 弹窗保存后，项目列表和当前项目信息刷新。

## 4. 项目内容调整弹窗

项目内容调整不跳转新页面，使用弹窗。

弹窗字段建议：

- 项目名称
- 客户名称
- 项目备注
- 项目状态

弹窗按钮：

- 取消
- 保存

保存后：

- 更新项目信息。
- 保持当前页面不变。
- 当前选中的项目仍然选中。

## 5. 导入导出页面

导入导出作为左侧第二个栏目。

功能包括：

- 导入手机采集包
- 导出给手机同步包
- 导出 Excel
- 查看最近导入记录
- 查看最近导出记录
- 自动备份提示

## 6. 设计原则

- 项目是电脑端管理的主入口。
- 样品不作为单独导航入口。
- 样品必须归属于项目查看。
- 项目调整使用弹窗，避免页面跳转。
- 高频操作可以放在项目页面顶部。
