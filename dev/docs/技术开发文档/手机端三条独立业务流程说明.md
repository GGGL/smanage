# 手机端三条独立业务流程说明

## 1. 总体原则

手机端首页只有三个主入口：

- 录入
- 查看
- 同步

这三个入口是独立业务路径，不互相强行跳转。

## 2. 路径一：录入

流程：

```text
首页
→ 录入
→ 录入详情
→ 结束
```

业务说明：

- 录入路径只负责新增采集。
- 用户拍照、填写图片名称、规格、产地、价格、备注、项目。
- 点击确认后保存到手机本地。
- 保存成功后进入“录入详情页”。
- 录入详情页只展示刚刚保存的记录。
- 录入详情页可提供：
  - 再录一条
  - 完成
- 点击完成后结束本次录入业务，可返回首页。

## 3. 路径二：查看

流程：

```text
首页
→ 查看
→ 项目列表
→ 项目详情
→ 具体产品详情
→ 结束
```

业务说明：

- 查看路径用于浏览和管理已有数据。
- 项目列表展示所有项目。
- 点击项目后进入项目详情页。
- 项目详情页展示该项目下所有产品/样品。
- 点击某个产品后进入具体产品详情页。
- 具体产品详情页允许调整数据，例如：
  - 图片名称
  - 规格
  - 产地
  - 价格
  - 备注
  - 状态
- 调整保存后，仍然属于查看管理路径，不跳回录入路径。
- 完成查看或调整后结束本次查看业务，可返回首页或项目列表。

## 4. 路径三：同步

流程：

```text
首页
→ 同步中心
→ 同步操作
→ 结束
```

业务说明：

- 同步中心只处理数据包导入和导出。
- 导出给电脑：
  - 生成完整 zip 数据包。
  - 弹出系统分享面板。
  - 用户传给电脑端。
- 导入电脑同步包：
  - 选择电脑端生成的 zip 包。
  - 手机端解析并合并本地数据。
  - 显示同步结果。
- 同步完成后结束本次同步业务。

## 5. 页面关系

```text
首页
├─ 录入
│  └─ 录入详情
│     └─ 结束
├─ 查看
│  └─ 项目列表
│     └─ 项目详情
│        └─ 具体产品详情（可调整）
│           └─ 结束
└─ 同步中心
   └─ 同步
      └─ 结束
```

## 6. 开发注意事项

- 录入详情页和具体产品详情页不是同一个业务页面。
- 录入详情页用于确认刚新增的记录。
- 具体产品详情页用于查看和调整已有记录。
- 同步中心不要混入查看和录入功能。
- 三条路径都可以返回首页，但业务状态要各自独立。
