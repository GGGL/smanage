# 样品采集管理系统：业务流程与技术栈开发文档

## 1. 项目定位

本系统用于个人场景下的产品样品采集、整理和管理。

核心目标：

- 手机端负责现场采集：拍照、录入、查看本地数据、导入导出同步包。
- 电脑端负责后期管理：导入手机包、按项目查看样品、调整样品内容、导出同步包。
- 手机和电脑不依赖实时联网，不需要云服务器。
- 手机端是 Web 应用打包成 App，可安装到 Android/iOS。
- 电脑端是本地 Web 管理程序，双击启动后自动打开浏览器。
- 数据通过 zip 包双向同步，手机和电脑都保留本地数据。

整体循环：

```text
手机端采集记录
→ 手机端导出数据包
→ 电脑端导入数据包
→ 电脑端按项目管理和修改
→ 电脑端导出同步包
→ 手机端导入同步包更新
→ 手机继续采集
```

## 2. 手机端业务流程

手机端首页只有三个主入口：

- 录入
- 查看
- 同步

三条路径相互独立，不互相强行跳转。

### 2.1 路径一：录入

```text
首页
→ 录入
→ 录入详情
→ 结束
```

说明：

- 录入路径只负责新增采集。
- 用户拍照、填写图片名称、规格、产地、价格、备注、项目。
- 点击确认后保存到手机本地。
- 保存成功后进入“录入详情页”。
- 录入详情页只展示刚刚保存的记录。
- 录入详情页提供“再录一条”和“完成”。
- 点击完成后结束本次录入业务，可返回首页。

### 2.2 路径二：查看

```text
首页
→ 查看
→ 项目列表
→ 项目详情
→ 具体产品详情
→ 结束
```

说明：

- 查看路径用于浏览和调整已有数据。
- 项目列表展示手机本地所有项目。
- 点击项目后进入项目详情页。
- 项目详情页展示该项目下的产品/样品。
- 点击某个产品后进入具体产品详情页。
- 具体产品详情页允许调整图片名称、规格、产地、价格、备注、状态。
- 调整保存后仍属于查看路径，不跳回录入路径。

### 2.3 路径三：同步

```text
首页
→ 同步中心
→ 同步操作
→ 结束
```

说明：

- 同步中心只处理数据包导入和导出。
- 导出给电脑：生成完整 zip 数据包并弹出系统分享面板。
- 导入电脑同步包：选择电脑端生成的 zip 包，解析后合并本地数据。
- 同步完成后显示新增、更新、图片、冲突数量。

## 3. 电脑端业务流程

电脑端不采用一个页面展示全部数据，改为分层进入，避免后期数据量大时页面卡顿。

左侧栏目只保留：

- 项目
- 导入导出

### 3.1 项目管理路径

```text
项目列表
→ 点击某个项目
→ 项目详情页
→ 点击项目内某个样品
→ 调整样品具体内容
```

### 3.2 页面一：项目列表

只显示项目汇总，不加载样品详情。

显示内容：

- 项目名称
- 样品数量
- 最近更新时间
- 当前状态

操作：

- 搜索项目
- 新建项目
- 导入数据包
- 点击项目进入项目详情

### 3.3 页面二：项目详情

进入某个项目后，只加载该项目下的样品。

显示内容：

- 当前项目名称
- 当前项目基础信息
- 当前项目下的样品列表
- 样品缩略图
- 样品名称、规格、状态等摘要

操作：

- 返回项目列表
- 调整项目内容
- 搜索当前项目内样品
- 点击样品进入样品调整页
- 导出当前项目 Excel

### 3.4 页面三：调整样品具体内容

点击某个样品后进入样品调整页。

显示和编辑内容：

- 图片
- 图片名称
- 规格
- 产地
- 价格
- 状态
- 备注

操作：

- 保存
- 取消
- 返回项目详情

### 3.5 导入导出页面

导入导出作为左侧第二个栏目。

功能：

- 导入手机采集包
- 导出给手机同步包
- 导出 Excel
- 查看最近导入记录
- 查看最近导出记录
- 自动备份提示

## 4. 性能和加载策略

电脑端必须按层级加载数据。

```text
打开项目列表：
加载 projects summary

点击项目：
加载 project detail + samples summary by projectId

点击样品：
加载 sample detail by sampleId

保存样品：
更新 sample，返回项目详情或停留当前页
```

原则：

- 项目列表页只加载项目汇总。
- 项目详情页只加载当前项目的样品。
- 样品调整页只加载当前样品完整数据和大图。
- 项目详情页图片只显示缩略图。
- 不在同一页面同时加载全部项目、全部样品、全部图片和全部详情。

## 5. 推荐技术栈

### 5.1 手机端

手机端采用“Web 应用 + App 容器打包”的方式。

```text
Vue 3
Vite
Capacitor
TypeScript
IndexedDB / Dexie.js
JSZip
SheetJS
Capacitor Camera
Capacitor Filesystem
Capacitor Share
Capacitor File Picker
```

说明：

- Vue 3：负责手机端界面和业务交互。
- Vite：负责前端开发和打包。
- Capacitor：把 Web 应用打包成 Android/iOS App。
- IndexedDB + Dexie.js：手机端本地存储。
- JSZip：生成和读取 zip 数据包。
- SheetJS：如需手机端导出 Excel，可使用。
- Camera：调用手机相机拍照。
- Filesystem：读写本地文件。
- Share：导出后弹出系统分享面板。
- File Picker：导入电脑端同步包。

平台支持：

- Android：可在 Windows 上使用 Android Studio 打 APK。
- iOS：需要 Mac、Xcode 和 Apple Developer 相关环境。

### 5.2 电脑端

电脑端采用“本地服务 + 浏览器界面 + 可执行程序打包”的方式。

```text
Python FastAPI
SQLite
Vue 3
openpyxl
zipfile
PyInstaller
本地图片目录
自动打开浏览器
```

说明：

- FastAPI：提供本地接口服务。
- SQLite：电脑端本地数据库。
- Vue 3：电脑端管理界面。
- openpyxl：导出 Excel。
- zipfile：导入和导出同步包。
- PyInstaller：打包成 Windows 可执行程序。
- 本地图片目录：保存样品照片和缩略图。
- 自动打开浏览器：双击程序后自动进入管理页面。

电脑端最终使用方式：

```text
双击 样品管理.exe
→ 自动启动本地服务
→ 自动打开浏览器
→ 管理样品数据
```

## 6. 环境隔离策略

开发阶段必须使用 dev 环境，正式发布时再切 production 环境，避免开发 APK、测试数据包和正式数据互相覆盖。

### 6.1 手机端环境

开发版：

```text
应用名：样品采集 Dev
包名：com.gggl.smanage.dev
环境标识：dev
本地数据库：sample-mobile-db-dev
导出文件名：sample_sync_dev_时间.zip
```

正式版：

```text
应用名：样品采集
包名：com.gggl.smanage
环境标识：production
本地数据库：sample-mobile-db
导出文件名：sample_sync_时间.zip
```

规则：

- Android/iOS 判断是否同一个 App，核心看包名。
- dev 包名和 production 包名必须不同。
- dev App 和正式 App 可以同时安装。
- dev App 的本地数据不能和正式 App 共用。
- dev 同步包必须带 `appEnv: "dev"`。

### 6.2 电脑端环境

开发版数据目录：

```text
dev/desktop/app/data-dev/
```

正式版数据目录：

```text
用户目录/样品管理/data/
```

规则：

- 开发阶段只写入 `data-dev`。
- 正式版不使用项目源码目录里的测试数据。
- 导入数据包时读取 `manifest.appEnv`。
- dev 包导入 production 时需要明确提示。
- production 包导入 dev 可以允许，用于问题复现和测试。

### 6.3 环境配置建议

建议统一配置：

```text
APP_ENV=dev 或 production
APP_NAME=样品采集 Dev 或 样品采集
APP_ID=com.gggl.smanage.dev 或 com.gggl.smanage
DATA_DIR=data-dev 或正式数据目录
```

开发默认值必须是：

```text
APP_ENV=dev
```

正式打包时再显式切换：

```text
APP_ENV=production
```

```json
{
  "packageId": "PKG-20260710-153000-mobile",
  "packageType": "mobile_to_desktop",
  "appEnv": "dev",
  "sourceDevice": "mobile",
  "exportedAt": "2026-07-10T15:30:00+08:00",
  "schemaVersion": 1,
  "appVersion": "1.0.0"

```

packageType 可选：

```text
mobile_to_desktop

```

appEnv 可选：

```text
dev
```

导入规则：

- dev 环境导出的包，文件名必须包含 `_dev_`。
- production 环境导出的包，不包含 `_dev_`。
- 电脑端导入时应显示包来源环境。
- 正式环境导入 dev 包时需要二次确认。

## 8. 同步合并规则

因为电脑端会反向修改手机数据，所以不能简单覆盖，必须做合并。

基础规则：

- 每个项目和样品必须有唯一 id。
- 手机和电脑都保存完整数据。
- 导入数据包时，根据 id 判断新增或更新。
- 使用 updatedAt 和 version 判断新旧。
- 删除采用 deleted 标记，不直接物理删除。
- 导入前自动备份当前本地数据。

### 8.1 手机包导入电脑

- 电脑端不存在该记录：新增。
- 电脑端存在该记录，手机版本更新：更新。
- 电脑端存在该记录，电脑版本更新：保留电脑版本。
- 两边都修改过同一条：生成冲突记录，默认保留电脑版本。
- 图片不存在则复制导入，已存在则跳过。

### 8.2 电脑包导入手机

- 手机端不存在该记录：新增。
- 手机端存在该记录，电脑版本更新：更新。
- 手机端存在该记录，手机版本更新：保留手机版本或提示冲突。
- 默认策略：电脑端修改优先，因为电脑端是整理和确认中心。

### 8.3 冲突处理

第一版建议简单处理：

- 默认保留电脑端版本。
- 手机端版本另存为冲突副本。
- 在电脑端显示“冲突待确认”。
- 用户可手动选择保留哪一版。

## 10. 开发阶段规划

### 第一阶段：核心闭环

目标：跑通手机采集、导出、电脑导入、电脑修改、导出、手机导入。

内容：

- 手机端录入路径：录入 → 录入详情 → 结束。
- 手机端查看路径：查看 → 项目列表 → 项目详情 → 具体产品详情。
- 手机端同步路径：同步中心 → 导出/导入。
- 电脑端项目列表页，只加载项目汇总。
- 电脑端项目详情页，只加载当前项目样品。
- 电脑端样品调整页，只加载当前样品详情。
- 双向完整 zip 数据包导入导出。
- dev 环境隔离：手机端 dev 包名，电脑端 data-dev 目录，同步包 manifest 写入 appEnv。

### 第二阶段：管理效率

目标：让电脑端和手机端管理更顺手。

内容：

- 项目搜索。
- 项目内样品搜索。
- 样品状态管理。
- 缩略图生成和预览。
- Excel 导出。
- 导入日志。
- 冲突提示。

### 第三阶段：稳定性和打包

目标：让系统变成可长期使用的小工具。

内容：

- 手机端 Android APK 打包。
- 手机端 iOS 适配准备。
- 电脑端 PyInstaller 打包。
- 自动备份。
- 数据恢复。
- 异常包校验。
- 大图片压缩和缩略图优化。

## 11. 第一版取舍

第一版建议做：

- 手机端使用 Vue 3 + Capacitor。
- 手机端本地存储使用 IndexedDB + Dexie.js。
- 手机端开发版使用 `com.gggl.smanage.dev`。
- 电脑端使用 FastAPI + SQLite。
- 电脑端开发版使用 `dev/desktop/app/data-dev/`。
- 手机端导出完整包，不做增量包。
- 电脑端导出完整同步包。
- 同步包写入 `appEnv`。
- 电脑端按项目分层加载，不做全量大列表。
- 冲突时默认电脑端优先。
- 删除只做 deleted 标记。

第一版暂不做：

- 云服务器。
- 多用户权限。
- 实时同步。
- 复杂审批。
- 增量同步。
- 消息通知。
- 一页展示全部项目和全部样品。
- 开发版和正式版共用同一个包名。
- 开发版和正式版共用同一个数据目录。

## 12. 风险和注意事项

- iOS 打包需要 Mac 和 Xcode，不能只靠 Windows 完成正式 iOS 打包。
- 手机端文件导入导出在 Android 和 iOS 上体验不同，要通过 Capacitor 插件统一封装。
- 图片文件可能较大，导出 zip 前应压缩图片或生成缩略图。
- 不能用简单覆盖同步，否则电脑端修改可能被手机数据覆盖。
- 每次导入前，电脑端应自动备份当前数据库。
- 每次手机导入同步包前，也应先生成本地备份。
- 电脑端不能在项目列表页一次性加载所有样品和图片。
- 开发 APK 和正式 APK 不能使用同一个包名，否则会互相覆盖或因签名不同安装失败。
- 开发版同步包不能无提示导入正式环境，否则可能污染正式数据。

## 13. 最终推荐方案

手机端：

```text
Vue 3 + Vite + Capacitor + Dexie.js + JSZip + Camera/Filesystem/Share/File Picker
```

电脑端：

```text
FastAPI + SQLite + Vue 3 + openpyxl + zipfile + PyInstaller
```

同步方式：

```text
完整 zip 数据包双向同步
电脑端优先处理冲突
导入前自动备份
删除使用软删除
图片生成缩略图
电脑端按项目分层加载
dev 和 production 环境隔离
```

这套方案适合个人离线使用，开发成本可控，并且避免后期数据量增加导致电脑端主页面卡顿。
