# 电脑端分层页面交互说明

## 1. 核心交互

电脑端不采用一个页面展示全部数据，改为分层进入：

```text
项目列表
→ 项目详情
→ 调整样品具体内容
```

## 2. 左侧栏目

左侧只保留：

- 项目
- 导入导出

## 3. 页面一：项目列表

只显示项目汇总，不加载样品详情。

显示内容：

- 项目名称
- 样品数量
- 最近更新时间
- 当前状态

操作：

- 搜索项目
- 新建项目
- 导入数据包
- 点击项目进入项目详情

## 4. 页面二：项目详情

进入某个项目后，只加载该项目下的样品。

显示内容：

- 当前项目名称
- 当前项目基础信息
- 当前项目下的样品列表
- 样品缩略图
- 样品名称、规格、状态等摘要

操作：

- 返回项目列表
- 调整项目内容
- 搜索当前项目内样品
- 点击样品进入样品调整页
- 导出当前项目 Excel

## 5. 页面三：调整样品具体内容

点击某个样品后进入样品调整页。

显示和编辑内容：

- 图片
- 图片名称
- 规格
- 产地
- 价格
- 状态
- 备注

操作：

- 保存
- 取消
- 返回项目详情

## 6. 性能原则

- 项目列表页只加载项目汇总。
- 项目详情页只加载当前项目的样品。
- 样品调整页只加载当前样品的完整数据和大图。
- 项目详情页图片只显示缩略图。
- 不在同一个页面同时加载全部项目、全部样品和全部图片。

## 7. 推荐数据加载方式

```text
打开项目列表：
加载 projects summary

点击项目：
加载 project detail + samples summary by projectId

点击样品：
加载 sample detail by sampleId

保存样品：
更新 sample，返回项目详情或停留当前页
```

这样后期样品数量变多时，页面仍然会比较稳。
